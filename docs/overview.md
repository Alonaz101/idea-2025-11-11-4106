# Overview

This repository contains the codebase for the mood-based recipe recommendation application as described by Jira issues SCRUM-334 to SCRUM-341.

## Features by Jira Issue

- **SCRUM-334:** MVP Architecture with React frontend for mood input and recipe display, Node.js/Express backend, MongoDB data store.
- **SCRUM-335:** Data models for User, Recipe, Feedback and API specs for recommendations, recipes, and feedback.
- **SCRUM-336:** Security (HTTPS enforced, GDPR prep), performance (stateless backend, connection pooling) and basic monitoring.
- **SCRUM-337:** Post-MVP user management (OAuth, JWT), user preferences, mood history, and advanced recipe filtering.
- **SCRUM-338:** Integration with mood tracking APIs and social sharing of recipes.
- **SCRUM-339:** Future AI-powered recommendation engine and microservices architecture.
- **SCRUM-340:** Real-time collaboration features support like live chat and shared cooking.

## Architecture

- Backend in Node.js/Express with MongoDB.
- Frontend in React.
- REST API endpoints defined.

This codebase represents the initial MVP and foundation for future enhancements.
